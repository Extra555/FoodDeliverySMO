# Food Delivery SMO — Вариант 9

## Описание

Программа моделирует систему массового обслуживания для центра обработки заказов доставки еды.  
В системе несколько ресторанов генерируют заказы, которые обрабатываются операторами с учётом заданных дисциплин обслуживания.

Поддерживаются:
- пошаговый режим моделирования;
- автоматический режим до заданного момента времени;
- вывод календаря событий;
- сбор статистики.

---

## Конфигурация системы

По умолчанию:

| Параметр                                  | Значение |
|-------------------------------------------|----------|
| Ресторанов                                | 3 |
| Операторов                                | 2 |
| Интервал генерации заказов  | 5.0 |
| Среднее время обработки                   | 3.0 |
| Вместимость буфера                        | 5 |

Все параметры можно изменить в начале `smo_food_center.py`.

---

## Реализованные дисциплины (вариант 9)

| Компонент | Дисциплина | Реализация |
|-----------|------------|------------|
| Источник | ИБ + И31 | Равномерный детерминированный интервал |
| Прибор | П32 | Экспоненциальное время обработки |
| Буфер | Д1ОЗ2 | FIFO, сдвиг при извлечении |
| Отказ | Д1ОО5 | Новый заказ отклоняется, буфер не изменяется |
| Выбор оператора | Д2П1 | Свободный оператор с минимальным номером |
| Выбор заявок | Д2Б5 | FIFO (корректно для варианта) |

---

## Основные компоненты программы

- **RestaurantSource** — генерирует заказы.
- **OrderBuffer** — хранит заказы по принципу FIFO.
- **Operator** — обрабатывает заявки, использует экспоненциальное распределение.
- **Event** — календарные события.
- **SMO System** — главный цикл симуляции.

---

## Режимы работы

### 1. Пошаговый режим
Позволяет:
- выполнять события по одному;
- наблюдать текущее состояние (буфер, операторы, рестораны).

### 2. Автоматический режим
Моделирование выполняется до указанного времени без задержек.

### 3. Календарь событий
Выводит список последних событий (ORDER_GENERATED, OPERATOR_FREE и т.д.).

---

## Статистика

После завершения моделирования выводится:

- всего заказов;
- обработано заказов;
- отклонено заказов;
- процент отказов;
- среднее время ожидания по каждому ресторану.
