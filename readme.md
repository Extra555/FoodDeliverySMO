# Food Delivery SMO System - Вариант 9

## Описание проекта

Система массового обслуживания (СМО) для центра обработки заказов доставки еды в реальном времени. Проект реализует имитационную модель обработки заказов от нескольких ресторанов с различными дисциплинами обслуживания.

## Технические характеристики варианта 9

| Компонент | Тип | Описание |
|-----------|-----|----------|
| **Источник** | ИБ + И31 | Бесконечный источник с равномерным распределением |
| **Прибор** | П32 | Экспоненциальное время обработки |
| **Буферизация** | Д1032 | FIFO со сдвигом заказов |
| **Отказ** | Д1005 | Отказ новой заявки без изменения буфера |
| **Выбор прибора** | Д2П1 | Приоритет по номеру оператора |
| **Выбор заявок** | Д2Б5 | Пакетная обработка по ресторанам |

## Архитектура системы

### Основные компоненты

- **Рестораны (RestaurantSource)** - генерируют заказы через равные промежутки времени
- **Операторы (Operator)** - обрабатывают заказы с экспоненциальным распределением времени
- **Диспетчеры (Dispatcher)** - управляют потоком заказов:
  - `PlacementDispatcher` - распределение новых заказов
  - `SelectionDispatcher` - выбор заказов из буфера

### Конфигурация системы по умолчанию

- **Количество ресторанов**: 3
- **Количество операторов**: 2
- **Интервал генерации заказов**: 5.0 единиц времени
- **Среднее время обработки**: 3.0 единиц времени

## Дисциплины работы

- **Д1032** - Заказы добавляются в конец очереди, при удалении происходит сдвиг остальных
- **Д1005** - При полном буфере новые заказы получают отказ без изменения состояния буфера
- **Д2П1** - Свободные операторы выбираются по приоритету номеров (от меньшего к большему)
- **Д2Б5** - Заказы обрабатываются пакетами по ресторанам
